Scrolling Alternatives

Cards vs Scrolls

Prior to web browser development, there was some discussion about what the best way to present nodes of information in a hypertext environment. One group, labelled the "Holy Scrollers" suggested that nodes with more information then can be seen at one time should scroll linearly up and down to provide access to this content. New information could be accessed through 'stretchtext' links that expand new content in-place.

The other approach, who's proponents were given the catchy title "Card Sharks" provided an alternative influenced by existing systems like HyperCard where each node has a fixed size and layout. These "cards" provide limited content per node and are linked to one another to provide access to more content.

General info can be found from a discussion around ipad scrolling and carding - but be sure to follow this with Mark Bernstein's Counter. Not sure if all the negatives presented about the original post are valid - but it looks the original debate might have been more nuanced then at first glance.

An article from 1995 from Jakob Nielsen provides a less opinionated presentation of the two options in the context of their potentials in HTTP. The afore mentioned scrolling essay, by Jesper Kiledal, also provides a good overview.
